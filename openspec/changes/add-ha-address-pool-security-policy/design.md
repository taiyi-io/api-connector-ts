## Context

当前太一云API连接器（TypeScript版本）需要扩展以支持三个新功能：高可用(HA)、地址池管理和安全策略。这些功能已在控制节点API中实现，现在需要在客户端连接器中提供相应的支持，以确保前端应用能够充分利用这些企业级特性。

现有代码结构：
- `src/connector.ts` - 主连接器类
- `src/data-defines.ts` - 数据结构和接口
- `src/enums.ts` - 枚举定义
- `src/request-params.ts` - API请求/响应类型
- `src/request-forwarder.ts` - 底层API通信

## Goals / Non-Goals

**Goals:**

- 实现高可用(HA)相关API方法，支持资源池HA开关控制和云主机HA状态管理
- 实现地址池管理API，支持四集合模型(ExternalV4/ExternalV6/InternalV4/InternalV6)的完整操作
- 实现安全策略组管理API，支持系统级策略模板和云主机级策略副本
- 保持与现有API的向后兼容性
- 提供完整的TypeScript类型定义和文档

**Non-Goals:**

- 实现控制节点端的业务逻辑（如HA切换、地址分配算法）
- 处理网络故障或断联检测（由控制节点负责）
- 实现DHCP服务器集成（不在范围内）
- 实现防火墙NAT转发规则管理（第二期实现）

## Decisions

### D1: API方法命名和结构

**选择**: 沿用现有的方法命名模式，使用 `tryXXX` + `waitTask` 模式处理异步操作。

**理由**: 保持API风格一致性，便于用户理解和使用。例如：
- `tryCreateAddressPool` + `waitTask`
- `tryCreateSecurityPolicy` + `waitTask`

**替代方案**: 为新功能创建不同的命名模式，但这会增加学习成本和代码复杂度。

### D2: 数据结构设计

**选择**: 严格按照控制API文档定义的数据结构，使用snake_case字段名。

**理由**: 确保与控制节点API的兼容性，减少数据转换错误。

**替代方案**: 使用camelCase字段名并在通信时转换，但这会增加代码复杂度和潜在的转换错误。

### D3: 错误处理

**选择**: 所有API方法返回 `BackendResult<T>` 类型，统一错误处理模式。

**理由**: 保持与现有API的一致性，便于用户统一处理错误。

**替代方案**: 为新功能使用不同的错误处理模式，但这会增加用户的学习成本。

### D4: 类型定义组织

**选择**: 在 `data-defines.ts` 中添加新的接口定义，在 `enums.ts` 中添加新的枚举类型。

**理由**: 保持代码结构的一致性，便于维护和查找。

**替代方案**: 创建新的文件来组织新的类型定义，但这会增加文件数量和复杂度。

### D5: 安全策略实现

**选择**: 支持系统级安全策略组和云主机级策略副本的完整生命周期管理。

**理由**: 与控制节点API保持一致，确保功能的完整性。

**替代方案**: 只实现部分安全策略功能，但这会限制用户的使用场景。

## Risks / Trade-offs

**[API兼容性风险]** → 控制节点API可能会有版本变化，需要确保连接器能够适应这些变化。
**[实现复杂度]** → 新功能涉及多个模块的修改，需要确保代码的可维护性。
**[测试覆盖]** → 新功能需要充分的测试覆盖，以确保稳定性。
**[文档同步]** → 需要确保API文档与实现保持同步，避免用户困惑。
